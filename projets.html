Supervision d'Infrastructure

Prometheus & Grafana

📥 Télécharger la documentation technique
01

🎯 Contexte & Objectifs

L'objectif de ce projet est de concevoir, déployer et sécuriser une solution de supervision centralisée (Prometheus) couplée à un outil de visualisation (Grafana)[cite: 2].

  • Cibles : Routeur Ubuntu, Hyperviseur PVE2, Switch Cisco 2960, Borne Wi-Fi Cisco Aironet 1702[cite: 2].
  • Métriques : Charge CPU, RAM, latences réseau, métriques DNS[cite: 2].
02

⚙️ Prometheus (Installation)

L'installation est réalisée manuellement via l'archive officielle Linux amd64 pour bénéficier de la version de production la plus récente[cite: 2].

  • Extraction : Création des répertoires systèmes /etc/prometheus et copie des binaires[cite: 2].
  • Démon Systemd : Création du fichier prometheus.service pour assurer la continuité de service au démarrage[cite: 2].
03

📈 Grafana & Sécurisation SSL

Installation de Grafana via les dépôts APT officiels avec clé GPG[cite: 2].

  • HTTPS : Génération d'un bi-clé et d'un certificat auto-signé dédié à Grafana[cite: 2].
  • Résolution d'incident : Correction de l'erreur SSL_ERROR_RX_RECORD_TOO_LONG en attribuant les privilèges de lecture restrictifs (640) au groupe grafana sur les certificats[cite: 2].
04

📡 Configuration YAML & Agents

Le fichier prometheus.yml centralise les jobs de requêtage[cite: 2].

  • Node Exporter : Déployé sur le Routeur Linux (port 9100) pour l'état système brut[cite: 2].
  • Proxmox Exporter : Utilitaire en Python interrogeant l'API interne de PVE2 (port 9221)[cite: 2].
05

🖧 Supervision SNMP (Cisco)

Acquisition des métriques réseau via le protocole standardisé SNMPv2c[cite: 2].

  • Configuration : Création d'une communauté SNMP en lecture seule team3 sur le Switch 2960 et la Borne AP[cite: 2].
  • SNMP Exporter : Ajout des jobs dans Prometheus avec un traducteur local (port 9116)[cite: 2].
06

✅ Recette & Validation

La phase de recette valide l'étanchéité de la collecte[cite: 2].

  • Prometheus : Vérification du statut nominal vert (UP) pour toutes les cibles sur /targets[cite: 2].
  • Grafana : Importation de tableaux de bord et vérification de la connexion HTTPS avec cadenas de sécurité[cite: 2].

Gestion de Parc & Ticketing

Déploiement GLPI & Active Directory

📥 Télécharger le dossier complet
01

🎯 Contexte & Objectifs

L'objectif de ce projet est de concevoir et de déployer une solution centralisée de gestion de parc informatique et de Service Desk au sein de la M2L[cite: 1].

  • Authentification : Interconnexion avec l'Active Directory via LDAP[cite: 1].
  • Automatisation : Inventaire du parc via le déploiement de GLPI Agent[cite: 1].
02

🖥️ Active Directory (SRV-AD01)

Mise en place de l'annuaire sur Windows Server 2019 (172.16.3.15)[cite: 1].

  • Installation : Ajout du rôle AD DS et des outils d'administration distants (RSAT)[cite: 1].
  • Promotion : Création de la nouvelle forêt racine WINTER.LOCAL et intégration du rôle Serveur DNS[cite: 1].
03

🔗 Jonction au Domaine

Intégration des postes clients au domaine WINTER.LOCAL[cite: 1].

  • Client Windows 11 : Modification du DNS et bascule du groupe de travail vers le domaine[cite: 1].
  • Client Ubuntu Desktop : Installation de realmd et sssd, exécution de realm join, et configuration PAM pour la création dynamique des répertoires /home/[cite: 1].
04

🪪 Interconnexion LDAP & GLPI

Liaison de l'application GLPI (172.16.3.39) à l'annuaire de manière sécurisée[cite: 1].

  • Remontée : Utilisation de l'attribut memberOf pour lire les groupes de sécurité Windows[cite: 1].
  • Automatisation : Règle d'affectation dynamique associant le groupe AD (ex: GG-GLPI-Techniciens) au profil GLPI Technician[cite: 1].
05

📦 Déploiement GLPI Agent

Automatisation de l'inventaire matériel et logiciel[cite: 1].

  • Environnement Windows : Installation via package .msi en tant que service d'arrière-plan (Windows Service) pointant vers l'URL du serveur[cite: 1].
  • Environnement Linux : Ajout de la clé GPG, installation via APT, édition de agent.cfg et activation via Systemd[cite: 1].
06

🔐 Sécurisation HTTPS (Apache2)

Désactivation du protocole HTTP clair (port 80) au profit du HTTPS (port 443)[cite: 1].

  • Certificats : Génération d'un certificat SSL auto-signé valide 3 ans[cite: 1].
  • VirtualHost : Configuration de default-ssl.conf, activation des modules Apache (a2enmod ssl) et redirection permanente (HTTP vers HTTPS)[cite: 1].
  • Impact : Ajustement des cibles des agents GLPI (Remote Targets) vers la nouvelle URL sécurisée[cite: 1].
